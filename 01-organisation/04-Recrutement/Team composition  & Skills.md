# Team Composition & Skills

## 1. Justification du choix de l’équipe

### 1.1 Besoins du projet
- **Type de solution :**  
  Plateforme web permettant d'exploiter, visualiser et analyser des données politiques et citoyennes.

- **Technologies envisagées :**  
  - Frontend : HTML, CSS, JavaScript / framework frontend  
  - Backend : Python (API)  
  - Data : pipelines ETL et modèles IA  
  - Base de données : base relationnelle ou data warehouse  
  - Infrastructure : déploiement cloud / CI-CD

- **Contraintes (délai, complexité, etc.) :**  
  - Développement dans un cadre académique avec un temps limité  
  - Coordination entre plusieurs pôles techniques  
  - Besoin de structurer un projet complet (frontend, backend, data, IA, infrastructure)

---

### 1.2 Arguments professionnels

- **Complémentarité des compétences :**

  L’équipe a été construite pour couvrir les différents besoins techniques du projet :

  - Frontend et UX/UI pour l’interface utilisateur
  - Backend pour la logique applicative et les APIs
  - Data / IA pour l’analyse et le traitement des données
  - DevOps pour l’infrastructure et le déploiement
  - Gestion produit pour coordonner les développements

- **Expériences passées :**

  Certains membres ont déjà travaillé sur :
  - développement d’API
  - projets Python et data
  - UX/UI et développement frontend
  - architecture technique

- **Disponibilités :**

  Les membres sélectionnés ont montré une motivation forte et une volonté de s’investir prioritairement dans le projet POPolitics.

- **Adéquation avec les besoins du projet :**

  La structure de l’équipe permet d’assurer :
  - une organisation technique claire
  - une répartition des responsabilités
  - une bonne communication entre les pôles.

## 3. Gap de compétences & plan de montée en compétences

### Manques identifiés

- structuration complète d’un projet full-stack
- mise en place d’une architecture DevOps
- intégration entre data, backend et frontend
- gestion de pipelines data et IA dans une application

---

### Actions prévues

Pour combler ces écarts :

- consultation de **documentations techniques**
- réalisation de **proof of concept (POC)**
- partage de ressources et tutoriels entre membres
- travail collaboratif et entraide dans l’équipe
- mise en place d’une documentation projet

Ces actions permettront à l’équipe de **monter en compétences progressivement tout en développant le projet**.
